## 观察结论

- 如果“关系事件命中率”持续偏低，优先补充事件 `narrative_tags` 中的关系标签。
- 如果“里程碑有变化但事件无反馈”，优先检查关系窗口命中池是否缺少对应标签事件。
- 如果全程无阶段变化，可提升冲突/和解相关选项的选择策略强度。
- 如果关键事件入口出现但“被选择/结算”很低，优先检查前端可选项展示顺序与文案可理解性。
- 如果 `render_source` 连续出现 `expression_fallback_*`，优先检查模型可用性/网络与 JSON 输出稳定性。
- 如果关系链事件命中低，优先放宽链路事件的 `day_min/relation_stage_any/weekly_rhythm_any` 条件。
- 如果链路事件命中高但完成旗标低，优先调整关键选项文案与测试脚本选项策略。
